Project Structure

Create a folder called docker-project with the following structure. The application is a task manager called TaskFlow.

docker-project/ index.html ← main page │ ├── style.css ← styles │ └── app.js ← JS logic ├── Dockerfile ← image recipe ├── nginx.conf ← server config ├── docker-compose.yml ← orchestration ├── .env.example ← env variables └── .dockerignore ← excluded files
💡
Why Nginx? Our application is 100% static (HTML/CSS/JS). Nginx is a lightweight and performant web server, perfect for serving this type of file inside a container.

Commands to create the structure:

Terminal
bash 
touch Dockerfile nginx.conf docker-compose.yml .dockerignore

Writing the Dockerfile

The Dockerfile is the recipe that describes how to build your Docker image. Each instruction creates a layer in the final image.

Dockerfile
dockerfile 
# ── Step 1: base image ───────────────────────────
# nginx:alpine = Nginx + Alpine Linux (~7 MB only)
FROM nginx:alpine

# ── Step 2: metadata ─────────────────────────────
LABEL maintainer="your.email@example.com"
LABEL version="1.0"
LABEL description="TaskFlow — Static Nginx Application"

# ── Step 3: copy app files ───────────────────────
COPY / /usr/share/nginx/html/

# ── Step 4: copy Nginx config ────────────────────
COPY nginx.conf /etc/nginx/conf.d/default.conf

# ── Step 5: port exposed by the container ────────
EXPOSE 80

# ── Step 6: startup command ──────────────────────
CMD ["nginx", "-g", "daemon off;"]
Instruction Role
FROM Base image (starting point)
LABEL Metadata (author, version…)
COPY Copies local files → container
EXPOSE Declares the internal port (documentation)
CMD Command run when the container starts

Also create the Nginx configuration file:

nginx.conf
nginx 
server {
    listen      80;
    server_name localhost;
    root        /usr/share/nginx/html;
    index       index.html;

    # Gzip compression
    gzip on;
    gzip_types text/css application/javascript text/html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Static asset caching
    location ~* \.(css|js|png|ico)$ {
        expires 7d;
    }
}

Building the Image (docker build)

The docker build command reads the Dockerfile and creates a local Docker image. Run it from the docker-project folder.

Terminal
bash 
# Build the image and give it a name + tag
docker build -t taskflow:1.0 .

# Verify the image was created
docker images

# Inspect image metadata
docker inspect taskflow:1.0

# View layer history
docker history taskflow:1.0
⚠️
Tag syntax: name:version. If you omit the tag, Docker uses latest by default. Always use explicit versioning in production.

Expected output of docker images:

Output text

Running a Container (docker run)

A container is a running instance of an image. Here's how to start one from your taskflow:1.0 image.

Terminal
bash 
# Run the container in the background (-d)
# Map host port 3000 to internal port 80
docker run -d -p 3000:80 --name taskflow-app taskflow:1.0

# Check the container is running
docker ps

# Follow logs in real time
docker logs -f taskflow-app

# Open an interactive shell inside the container
docker exec -it taskflow-app sh

# Stop the container
docker stop taskflow-app

# Remove the container
docker rm taskflow-app
Option Meaning
-d Detached mode (background)
-p 3000:80 Host port 3000 → container port 80
--name Custom container name
-it Interactive mode with terminal
-f Follow logs in real time
✅
Open your browser at http://localhost:3000 to see the TaskFlow application running inside Docker!

Docker Compose

Docker Compose lets you define and run multiple services in a docker-compose.yml file. It's much more convenient than long docker run commands.

docker-compose.yml
yaml 
version: "3.9"

services:

  # ── Main service: the application ─────────────
  taskflow:
    build:
      context: .
      dockerfile: Dockerfile

    container_name: taskflow-app

    ports:
      - "${APP_PORT:-3000}:80"

    environment:
      - NODE_ENV=${NODE_ENV:-development}

    volumes:
      - taskflow-web:/usr/share/nginx/html

    restart: unless-stopped

    healthcheck:
      test: ["CMD", "wget", "-q", "--spider", "http://localhost"]
      interval: 30s
      timeout:  5s
      retries:  3

    networks:
      - taskflow-network
volumes:
  taskflow-web:
    driver: local

networks:
  taskflow-network:
    driver: bridge
Terminal — Compose Commands
bash 
# Start all services (auto build)
docker compose up -d

# Rebuild the image if the Dockerfile changed
docker compose up -d --build

# Check service status
docker compose ps

# View logs from all services
docker compose logs -f

# Stop services (without removing)
docker compose stop

# Stop AND remove containers
docker compose down

# Also remove volumes and images
docker compose down -v --rmi all

Environment Variables

Docker Compose can read a .env file to inject variables into your services. This is essential for separating configuration across environments (dev, prod).

.env.example → copy to .env
env 
# Port exposed on the host machine
APP_PORT=3000

# Environment: development | production
NODE_ENV=development
Terminal
bash 
# Create the .env from the example
cp .env.example .env

# Run with an inline variable override
# Run this command using git bash in the same project directory
APP_PORT=8080 docker compose up -d

# Inspect a container's environment variables
docker inspect --format '{{range .Config.Env}}{{.}}{{"\n"}}{{end}}' taskflow-app
🔐
Security: Always add .env to your .gitignore. Never commit secrets (passwords, API keys) in a Dockerfile or docker-compose.yml.

Volumes & Hot Reload

Volumes allow you to mount a folder from your machine into the container. Any change in / is immediately visible without rebuilding the image.

docker-compose.yml — volume section
yaml 
volumes:
  # Mount the source code (read-only inside the container)
  - ./html:/usr/share/nginx/html
  #  ^local folder  ^container folder  ^:ro = read-only

Try it live:

  • Run docker compose up -d
  • Open http://localhost:3000
  • Edit index.html (e.g. change the title)
  • Refresh the browser → the change is instant!
Terminal — Volume Management
bash 
# List all Docker volumes
docker volume ls

# Inspect a named volume
docker volume inspect <volume_name>

# Remove unused volumes
docker volume prune

Publishing to Docker Hub

Docker Hub is the official public registry. You can push your images there to share them or deploy them on a remote server.

Terminal
bash 
# 1. Log in to Docker Hub
docker login

# 2. Tag the image with your Docker Hub username
#    Format: yourusername/image-name:tag
docker tag taskflow:1.0 yourusername/taskflow:1.0

# 3. Push the image to the registry
docker push yourusername/taskflow:1.0

# 4. On another server: pull and run
docker pull yourusername/taskflow:1.0
docker run -d -p 80:80 yourusername/taskflow:1.0

Essential Commands — Cheat Sheet

Command Description
docker build -t name:tag . Build an image
docker images List local images
docker rmi name:tag Remove an image
docker run -d -p H:C name Run a container in the background
docker ps List running containers
docker ps -a All containers (including stopped)
docker stop id Stop a container
docker rm id Remove a container
docker exec -it id sh Open a shell in the container
docker logs -f id Follow logs in real time
docker compose up -d Start Compose services
docker compose down Stop and remove
docker system prune -a Clean everything (images, containers, volumes)
